# OCR Service - План развития (Development Plan)

## 📌 Текущий фокус
Разработка и стабилизация пайплайна распознавания паспортов с использованием ML моделей и интеграцией в общую экосистему.

---

## 📅 Ближайший спринт (Backlog)

### 1. Инфраструктура и Деплой
- [x] Настроить Docker & Docker Compose
- [x] Создать инструкции по безопасному деплою (`DEPLOY_INSTRUCTION_FOR_SERVER.md`)
- [ ] Интегрировать CI/CD Pipeline (GitHub Actions)
- [ ] Оптимизация размера Docker-образов для ML.

### 2. OCR Пайплайн
- [ ] Интеграция `PaddleOCR-VL` / `olmOCR-2`
- [ ] Модуль валидации MRZ
- [ ] Модуль восстановления ФИО (Patronymic Lexicon)
- [ ] Очередь задач (Redis + Celery/RQ) для асинхронной обработки изображений.

### 3. API и Мониторинг
- [ ] Healthcheck эндопоинт с выводом загрузки GPU/CPU
- [ ] Автоматизация метрик качества OCR
- [ ] Интеграция с S3 (MinIO) для хранения сканов паспортов.

---

## 🎯 Среднесрочные цели (Q2-Q3 2026)

### 1. Улучшение точности (Accuracy)
- Сбор размеченного датасета узбекских/российских паспортов.
- Fine-tuning моделей (LoRA) для сложных случаев (плохое освещение, блики).
- A/B тестирование новых моделей (`HunyuanOCR`, `DeepSeek-OCR`).

### 2. Архитектура
- Разделение API и Worker-нод на разные контейнеры.
- Интеграция LLM-агента для fall-back логики при низкой confidence от OCR.
- Настройка rate-limits.

---

## 🚀 Долгосрочные цели (Roadmap)
- Масштабирование сервиса (Kubernetes).
- Мобильное SDK для захвата паспортов с камеры.
- Поддержка новых типов документов (ID карты, водительские права).
